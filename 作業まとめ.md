# 凋叶棕 歌詞クイズ SPA — 作業まとめ

## プロジェクト概要

凋叶棕 (RD-Sounds) の楽曲における当て字・特殊読みを出題する日本語歌詞クイズ。
Vite + React (SPA)。データは YAML で管理、コード変更なしにアルバム追加可能。

- 公開先: `../luy_web/public/song_quiz/`（`npm run deploy` でコピー）
- 開発: `npm run dev`（localhost:5173）

## 凋叶棕ファンサイト風 UIリデザイン + correct index修正

### 問題点

1. デザインが「モダンダークテック」で凋叶棕の世界観と乖離（枯れ葉・秋・退廃・文語・和の美）
2. `correct` フィールドが 0-based index のため、YAML 手書き時にミスが無検知になる構造的問題

### ① correct index 構造的修正（QuizScreen.jsx）

| | コード |
|---|---|
| 変更前 | `choiceIndex === question.correct`（index 比較）|
| 変更後 | `question.choices[choiceIndex] === question.answer.text`（値比較）|

`handleChoice` / `getButtonClass` / `getFeedbackMessage` の3箇所を修正。
`question.choices[question.correct]` への参照を全廃し、`question.answer.text` / `question.answer.reading` のみ使用。
→ YAML の `correct` フィールドは残るが、ロジックが依存しなくなった。

## テキスト → YAML 自動生成スクリプト（完了）

### `scripts/gen-quiz-yaml.mjs` 実装

`サイトの内容.txt`（1563行、凋叶棕全ナンバリング作品）を自動解析し、クイズ YAML を生成。

**実行結果:**
- 40個のアルバムを新規生成（dyj-02, dyj-04～dyj-42）
- 422曲、597問のクイズデータを抽出
- 既存アルバム（dyj-01, dyj-03）は保護・上書きなし
- `public/data/albums.yaml` に新規エントリを追記

**処理フロー:**
1. テキストを空行で分割 → ブロック化（各ブロック = 1アルバム）
2. 正規表現で当て字抽出：`target（reading）` を解析
3. target の直前テキストを `prompt` に、行全体を `display` に変換
4. 全問の reading をひらがな/カタカナプールに分類
5. 各問につき、正解 + 同じ文字種から3つ選択肢を生成・シャッフル
6. ID割り当て（`dyj-02-01-q1` など）
7. YAML出力 + インデックス更新

**新規コンテンツ生成（テキストファイル更新時）:**
```bash

```

**アプリ起動:**
```bash
npm run dev
```

**本番ビルド・デプロイ:**
```bash
npm run build
```
